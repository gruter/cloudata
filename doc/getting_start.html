In this section, you can run cloudata on a single machine(Windows). And can create table, insert data, select data.

System Requirements

In order to run or compile cloudata you need to have following software already installed on your system

  • Java Virtual Machine: JDK6
  • Hadoop: 0.17 or higher. If you want to use other Distributed File System, implements CloudataFileSystem.
  • Zookeeper: http://hadoop.apache.org/zookeeper/

Download

You should have download a cloudata distribution and unzipped it. You can find following directories.

  • bin: contains cloudata shell scripts
  • conf: contains cloudata configuration files
  • docs: contains cloudata API docs
  • lib: contains related external libraries.
  • src: contains source files
  • build.properites
  • build.xml

Configure

1. Configure cloudata: change "conf/cloudata-site.xml"

<?xml version="1.0"?>
<configuration>
    <property>
        <name>cloudata.filesystem</name>
        <value>local</value>
        <description>local, hadoop</description>
    </property>

    <property>
        <name>cloudata.root</name>
        <value>C:/temp/cloudata_local</value>
        <description>root path for data file in DFS(Hadoop)</description>
    </property>

    <property>
        <name>cloudata.local.temp</name>
        <value>C:/temp/cloudata_local/temp</value>
        <description>A base for other temporary directories.</description>
    </property>

    <property>
        <name>changelog.data.dir</name>
        <value>C:/temp/cloudata_local/changelog</value>
        <description>path where change logs are stored.</description>
    </property>            
    
    <property>
        <name>changelog.image.dir</name>
        <value>C:/temp/cloudata_local/changelog/image</value>
        <description></description>
    </property>       
</configuration>
2. Configure zookeeper: change "conf/zoo.cfg" 
      	dataDir=C:/temp/cloudata/zookeeper/data
3. change "bin/cloudata.bat"
@echo off 
set JAVA_HOME="C:\Program Files\Java\jdk1.6"
set HADOOP_HOME="C:/download/hadoop-0.18.1"
...

Run Zookeeper

run window "command prompt"
cd ZOOKEEPER_HOME
java -cp .\zookeeper-3.1.1.jar;.\lib\log4j-1.2.15.jar;.\conf org.apache.zookeeper.server.quorum.QuorumPeerMain CLOUDATA_HOME\conf\zoo.cfg

Run LocalCluster

run window "command prompt"
C:\>cd C:\download\cloudata\bin
C:\download\cloudata\bin>cloudata org.cloudata.core.LocalCluster

Run Shell

1. Run shell 
run window "command prompt"
C:\>cd C:\download\cloudata\bin
C:\download\cloudata\bin>cloudata org.cloudata.core.client.Shell
2. Create table
create table T_TEST (col1, col2);
3. Show tables
show tables;
4. Insert data 
insert into T_TEST (col1, col2) values ( ( 'col1_ck1', 'col1_val1' ) , ( 'col2_ck1', 'col2_val1' ) ) where rowkey = 'rk1';
insert into T_TEST (col1, col2) values ( ( 'col1_ck2', 'col1_val2' ) , ( 'col2_ck1', 'col2_val2' ) ) where rowkey = 'rk1';
insert into T_TEST (col1, col2) values ( ( 'col1_ck2', 'col1_val2' ) , ( 'col2_ck1', 'col2_val2' ) ) where rowkey = 'rk3';
5. Select data
select * from T_TEST;

Run First App

Run FirstApp class in cloudata-examples.jar(You can see source in "src/examples/first_apps/org/cloudata/examples/first/FirstApp.java")
cloudata org.cloudata.examples.first.FirstApp
